# Aurum

**Aurum** — внутренний сервис AGIMA для создания, согласования, оплаты и сопровождения заявок на бюджет и затраты.

Сервис покрывает полный путь заявки:
- создание и черновики;
- согласование по ролям;
- валидацию конкурсных заявок через `HOD`;
- дополнительные согласования и отложенные решения;
- работу с квотами;
- теги и ручные списания по тегам;
- передачу в оплату;
- полную и частичную оплату;
- планирование цепочки платежей;
- комментарии, вложения, историю изменений и таймлайн.

## Что умеет Aurum

| Блок | Что есть в сервисе |
| --- | --- |
| Заявки | черновики, отправка на согласование, редактирование, закрытие и повторное открытие |
| Цеха и типы | `Аккаунтинг` с несколькими типами, `Транзит` с типом `Транзит`, остальные цеха с типом `Закупка` |
| Суммы | суммы без НДС и с НДС, автоматический пересчет, настраиваемая ставка НДС |
| Квоты | единая квота AGIMA, квоты цехов, квоты тегов, ручные списания без заявок, история изменений |
| Оплата | передача в оплату, планирование полного и частичных платежей, фиксация оплаты, отмена плана и платежа |
| Документы | вложения, drag-and-drop, PDF / Office / изображения / архивы |
| Аудит | комментарии, mentions, viewer access, история изменений, таймлайн событий и email-лог |

## Основные роли

| Роль | Зона ответственности |
| --- | --- |
| `AD` | создает и ведет свои заявки |
| `NBD` | согласует пресейлы и работает с квотой пресейлов |
| `AI-BOSS` | согласует AI-сценарии и смежные маршруты |
| `COO` | согласует заявки, видит все заявки, управляет тегами и квотами |
| `CFD` | финансовое согласование, теги, квоты, оплата |
| `BUH` | оплата, транши, курс валюты, финплан, теги, квоты |
| `HOD` | валидация и согласование заявок своего цеха, теги и квоты своего цеха |
| `ADMIN` | администрирование ролей и сопровождение нестандартных ситуаций |

## Ключевые пользовательские сценарии

### Обычная заявка

1. Автор создает заявку или сохраняет черновик.
2. Заполняет тип заявки, источник финансирования, суммы и обоснование.
3. Отправляет заявку на согласование.
4. После согласования передает заявку в оплату или закрывает без оплаты, если платеж не нужен.
5. После оплаты автор закрывает заявку.

### Конкурсное задание

1. Автор добавляет внутренних специалистов и подрядчиков.
2. По каждой строке указывает цех, часы и прямые затраты.
3. Сумма заявки собирается автоматически из прямых затрат.
4. Если нужна валидация цеха, заявка сначала идет к `HOD`.
5. После валидации заявка уходит в обычное согласование.

### Частичная оплата

1. `BUH` планирует сумму оплаты и дату платежа.
2. Если оплата идет частями, бухгалтер фиксирует каждую часть отдельно.
3. Для каждой части можно указать свою дату оплаты.
4. После фиксации части можно запланировать следующий платеж.
5. Когда остаток закрыт, заявка переводится в `Оплачено`.

### Квоты и теги

1. Общая квота AGIMA распределяется по цехам.
2. Квота цеха раскладывается по тегам.
3. Списание происходит по заявкам или вручную через `Потрачено без заявок`.
4. Все изменения попадают в историю квот.

## Технологический стек

**Frontend**
- Next.js 16
- React 19
- Tailwind CSS
- shadcn/ui

**Backend**
- Convex
- `@convex-dev/auth`

**Email**
- SMTP relay `agima.ru`

## Структура проекта

```text
src/app                страницы и сценарии UI
src/components         общие компоненты интерфейса
src/lib                клиентские хелперы, константы и бизнес-утилиты
convex                 схема, бизнес-логика, письма, квоты, согласования
docs                   продуктовая и пользовательская документация
public                 статические ресурсы
```

Ключевые backend-модули:
- `convex/requests.ts` — жизненный цикл заявки и оплата
- `convex/approvals.ts` — согласование
- `convex/quotas.ts` — квоты и лимиты
- `convex/emails.ts` — email-уведомления
- `convex/timeline.ts` — аудит событий

## Быстрый старт

### Требования

- Node.js 20+
- npm
- доступ к Convex deployment
- доступ к SMTP relay

### Установка

```bash
npm install
```

### Локальная разработка

Запустить Convex:

```bash
npx convex dev
```

Запустить frontend:

```bash
npm run dev
```

Открыть:
- [http://localhost:3000](http://localhost:3000)

### Полезные команды

```bash
npm test
npm run build
npx convex codegen
```

## Переменные окружения

Минимальный набор для `.env.local`:

```env
CONVEX_DEPLOYMENT=...
NEXT_PUBLIC_CONVEX_URL=...
NEXT_PUBLIC_CONVEX_SITE_URL=...

EMAIL_BASE_URL=http://localhost:3000
EMAIL_API_BASE_URL=https://public-host-for-email-endpoint
EMAIL_API_KEY=dev-email-key

SMTP_HOST=...
SMTP_PORT=...
SMTP_SERVERNAME=agima.ru
SMTP_FROM="Aurum <no-reply@aurum.agima.ru>"
SMTP_USER=
SMTP_PASS=

JWT_PRIVATE_KEY=...
JWKS=...
```

Важно:
- `EMAIL_API_BASE_URL` должен быть доступен из Convex.
- `EMAIL_BASE_URL` используется в ссылках внутри писем.
- неверные email или auth env ломают вход по коду и уведомления.
