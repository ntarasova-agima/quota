h1. Aurum: пользовательская инструкция

_Дата актуализации: 24.04.2026_

h2. Что такое Aurum

Aurum — внутренний сервис AGIMA для заявок на бюджет, согласований, оплаты, квот и рабочей коммуникации.

Через Aurum можно:
* создать заявку или черновик;
* отправить заявку на согласование;
* передать заявку в оплату;
* планировать и фиксировать платежи;
* работать с файлами, комментариями, тегами и историей изменений;
* контролировать квоты по AGIMA, цехам и тегам.

h2. Основные роли

* *AD* — автор заявок, комментарии, файлы, передача в оплату, закрытие заявки.
* *NBD*, *AI-BOSS*, *COO*, *CFD*, *BUH*, *HOD* — согласующие роли.
* *BUH* — также работает с оплатой, тегами и квотами.
* *CFD* — также работает с оплатой, тегами и квотами.
* *COO* — может смотреть все заявки, редактировать теги и квоты.
* *HOD* — руководитель цеха: видит и согласует свои заявки, валидирует прямые затраты, управляет квотами и тегами своего цеха.
* *ADMIN* — администрирование ролей и аварийные действия.

h2. Основные разделы

h3. Мои заявки

Здесь находятся все ваши заявки:
* черновики;
* активные заявки;
* закрытые и архивные заявки.

h3. На согласовании

Раздел для тех, от кого система ждет действие:
* согласование;
* валидацию цеха;
* планирование оплаты;
* фиксацию оплаты.

h3. Все заявки

Раздел для широких ролей и пользователей с доступом к просмотру:
* фильтрация по статусам, автору, тегу, типу заявки, категории и источнику;
* поиск по номеру заявки;
* фильтр по датам и по месяцу;
* просмотр всех доступных карточек.

h3. Справочник тегов

Используется для ведения рабочих тегов по цехам.

Теги могут пополнять:
* *CFD*
* *BUH*
* *COO*
* *ADMIN*
* *HOD* в пределах своих цехов

h3. Квоты

В Aurum используется единая квотная модель AGIMA:
* общая квота AGIMA;
* квоты по цехам;
* распределение квот по тегам;
* история изменений;
* ручное списание из тега без заявки.

h2. Как создать заявку

# Нажмите *Создать заявку*.
# Выберите *Цех*.
# Выберите *Тип заявки*.
# Выберите *Источник финансирования*.
# Заполните обязательные поля со звездочкой.
# При необходимости приложите файлы и договор.
# Сохраните черновик или отправьте на согласование.

Важно:
* незаполненные обязательные поля подсвечиваются красной рамкой;
* часть полей зависит от типа заявки, цеха и источника финансирования;
* если источник финансирования пока неизвестен, можно выбрать *Я не знаю*, а позже его уточнит *BUH* или *CFD*.

h2. Цеха и типы заявок

h3. Аккаунтинг

Доступны типы:
* Welcome-бонус
* Закупка
* Подарки
* Неформальное мероприятие
* Совместный мерч
* Конкурсное задание

h3. Транзит

Для цеха *Транзит* доступен тип:
* Транзит

Этот тип подставляется автоматически при выборе цеха.

h3. Остальные цеха

Для остальных цехов доступен тип:
* Закупка

h2. Источники финансирования

Основные источники:
* Квоты AGIMA
* Отгрузки проекта
* Я не знаю

Что важно:
* если выбраны *Отгрузки проекта*, заявка не списывается из квот;
* если выбраны *Квоты AGIMA*, списание происходит после присвоения тега;
* тег *Транзит* не участвует в квотах, если речь идет об отгрузках проекта.

h2. Ключевые поля в заявке

h3. На что нужен бюджет

Короткое название заявки на человеческом языке.

h3. Получатель сервиса

Поле используется там, где важен внутренний получатель сервиса или затрат.

Для типа *Закупка* здесь указывается:
* сотрудник;
* команда;
* отдел;
* внутренний получатель сервиса.

h3. Кому платим мы

Поле для контрагента или подрядчика. Показывается там, где нужен исходящий получатель платежа.

h3. Обоснование

Обязательное текстовое поле с ответом на вопрос, зачем нужен бюджет.

h3. Детали заявки

Необязательное широкое поле для:
* ссылок;
* описания предмета закупки;
* важного контекста;
* пояснений для согласующих и бухгалтерии.

h3. Связанные заявки

Сворачиваемый блок. Если заявка ни с чем не связана, его можно не заполнять.

h2. Суммы, НДС и валюта

В большинстве заявок есть поля:
* Сумма без НДС
* Сумма с НДС
* НДС
* Валюта

Как это работает:
* вы вводите ту сумму, которую знаете;
* второе поле рассчитывается автоматически;
* по умолчанию НДС равен *22%*;
* если поле НДС пустое, считается *0%*;
* для всех сумм разрешен ввод только чисел и разделителей.

h2. Предоплата

Если платеж нужно делить на предоплату и постоплату:
# включите *Требуется предоплата*;
# укажите сумму предоплаты без НДС или с НДС;
# укажите дату предоплаты.

Если предоплата не нужна, блок можно не заполнять.

h2. Договор и должная осмотрительность

Если сумма заявки больше *100 000* без НДС:
* обязателен *Договор с контрагентом*;
* можно указать ссылку, прикрепить файл или сделать и то и другое.

Если сумма заявки больше *500 000* без НДС:
* обязательна галочка *Проведена должная осмотрительность*;
* обязательна ссылка на задачу в Jira.

h2. Файлы

В блоке *Прикрепить файлы* можно загружать:
* PDF;
* документы Office;
* изображения;
* архивы zip, 7z, rar.

Ограничения:
* до 20 файлов;
* до 40 МБ на файл.

Отдельно можно прикреплять файлы договора с контрагентом.

h2. Особые сценарии

h3. Конкурсное задание

Для этого типа есть два блока:
* Внутренние специалисты
* Подрядчики

По каждой строке можно указать:
* имя;
* цех;
* часы;
* прямые затраты;
* *Валидация не требуется*.

Что важно:
* общая сумма считается автоматически по прямым затратам;
* строку может подтвердить *HOD* или *BUH*;
* если валидация нужна, заявка сначала попадает в *Ждет валидации цеха*.

h3. Транзит

Для типа *Транзит* используются дополнительные поля:
* Сумма отгрузки без НДС
* Сумма отгрузки с НДС
* Коэффициент транзита
* Дата отгрузки
* Когда платят нам
* ID и название отгрузки в финплане

*Коэффициент транзита* считается автоматически.

h2. Согласование

h3. Базовый маршрут

Для всех заявок обязателен *BUH*. Остальные роли добавляются по маршруту заявки.

h3. Дополнительное согласование

Согласующий может:
* *Согласовать и отправить дальше*
* *Отложить согласование и отправить дальше*

Что это дает:
* можно выбрать несколько дополнительных ролей;
* для *HOD* обязательно указывается конкретный цех;
* заявка получает статус *Частично согласовано*;
* автор получает письмо, кому заявка ушла дальше.

Если согласование было отложено, исходный согласующий остается в своих задачах и получает уведомление, когда дополнительный согласующий примет решение.

h3. Комментарии и доступ

В комментариях можно:
* упомянуть сотрудника по *@*;
* выдать доступ к просмотру заявки;
* обсуждать заявку даже тем, кто не входит в маршрут согласования.

Если сотрудника упомянули или ему выдали доступ:
* он видит заявку во *Всех заявках*;
* может открыть карточку и комментировать;
* не может редактировать заявку или согласовать ее, если у него нет таких прав.

h2. Оплата

Планирование оплаты доступно *BUH* и *CFD*.

В карточке заявки можно:
* запланировать оплату полностью;
* запланировать частичную оплату;
* добавить следующий плановый платеж;
* зафиксировать фактическую оплату;
* отменить план или отменить проведенный платеж.

h3. Полная оплата

Используется кнопка *Запланировать оплату*.

h3. Частичная оплата

Используется кнопка *Запланировать частичную оплату*.

После этого можно:
* добавить следующий плановый платеж;
* видеть нераспределенный остаток;
* фиксировать оплаты в той же ленте платежей.

Если *BUH* меняет сумму оплаты относительно исходной суммы заявки, автор получает письмо о том, кто и когда изменил сумму.

h2. Теги и квоты

Квоты устроены так:
* сверху находится общая квота AGIMA;
* ниже идут квоты цехов;
* внутри цеха квота распределяется по тегам.

Кто что видит:
* *CFD*, *BUH*, *COO*, *ADMIN* видят всю структуру;
* *HOD* видит только свой цех и его теги.

Кто что может менять:
* *CFD*, *BUH*, *COO*, *ADMIN* меняют общую квоту, квоты цехов и тегов;
* *HOD* меняет квоту и теги только своего цеха.

Дополнительно:
* у тега можно заполнить *Потрачено без заявок*;
* это ручное списание учитывается в остатке вместе со списаниями по заявкам;
* все такие изменения попадают в историю.

h2. Статусы заявки

* *Черновик* — заявка еще не отправлена.
* *Ждет валидации цеха* — нужно подтверждение прямых затрат.
* *Ожидает согласования* — заявка у согласующих ролей.
* *Частично согласовано* — часть ролей уже согласовала, но маршрут еще не завершен.
* *Согласовано* — маршрут согласования завершен.
* *Требуется оплата* — заявка передана в оплату.
* *Запланирована оплата* — указан следующий платеж.
* *Частично оплачено* — часть суммы уже проведена.
* *Оплачено* — платежи полностью завершены.
* *Заявка закрыта* — процесс по заявке завершен.
* *Отменена* — заявка снята из процесса.

h2. Что делает автор после согласования

Если заявка требует оплату:
# откройте карточку заявки;
# передайте ее в оплату.

Если оплата не требуется:
# откройте карточку заявки;
# закройте заявку без оплаты.
