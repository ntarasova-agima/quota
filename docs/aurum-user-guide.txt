h1. Aurum: пользовательская инструкция

_Дата актуализации: 09.04.2026_

h2. Что такое Aurum

Aurum — внутренний сервис AGIMA, через который сотрудники:
* создают заявки на бюджет и затраты;
* отправляют их на согласование;
* передают в оплату;
* отслеживают статусы;
* работают с файлами, комментариями и историей изменений.

h2. Основные разделы

h3. Мои заявки

Здесь находятся все заявки, которые вы создали:
* черновики;
* активные заявки;
* закрытые и архивные заявки.

Можно:
* искать заявку по номеру;
* фильтровать по статусу, типу и источнику;
* сортировать список;
* открывать карточку заявки.

h3. На согласовании

Рабочий раздел для согласующих ролей:
* NBD
* AI-BOSS
* COO
* CFD
* BUH
* HOD
* ADMIN

Здесь показываются заявки, по которым от вас ждут действия.

h3. Все заявки

Раздел доступен не всем ролям. В нем можно просматривать общий список заявок, фильтровать их и быстро переходить в карточки.

h3. Квоты

В зависимости от роли можно просматривать и редактировать квоты:
* пресейлы;
* внутренние затраты;
* AI-сервисы.

h3. Профиль

Здесь отображаются ваши данные и назначенные роли.

h2. Как создать заявку

# Нажмите *Создать заявку*.
# Выберите категорию заявки.
# Выберите источник финансирования.
# Заполните обязательные поля со звездочкой.
# При необходимости приложите файлы.
# Сохраните как черновик или отправьте на согласование.

Важно:
* если обязательное поле не заполнено, при сохранении оно подсветится красной рамкой;
* часть полей зависит от категории заявки и источника финансирования;
* некоторые связки категории и источника финансирования система ограничивает автоматически.

h2. Категории заявок

В сервисе доступны:
* Welcome-бонус
* Подарки
* Конкурсное задание
* Закупки сервисов (кроме AI)
* Сервисы/транзиты для клиентов
* AI-инструмент/подписка
* Неформальное мероприятие
* Совместный мерч

h2. Источники финансирования

Доступные источники:
* Отгрузки проекта
* Прибыль компании
* Квота на пресейлы
* Квоты на AI-инструменты
* Квота на внутренние затраты
* Я не знаю

Источник *Я не знаю* можно выбрать для любой категории.

h2. Как работают суммы и НДС

В большинстве заявок используются поля:
* Сумма без НДС
* Сумма с НДС
* НДС
* Валюта

Как это работает:
* вы вводите ту сумму, которую знаете;
* второе поле рассчитывается автоматически;
* по умолчанию НДС равен *22%*;
* если поле НДС пустое, считается *0%*;
* ставку НДС можно менять вручную.

h2. Вложения

В блоке *Прикрепить файлы* можно загрузить:
* PDF;
* документы Office;
* изображения;
* архивы zip, 7z, rar.

Ограничения:
* до 20 файлов;
* до 40 МБ на файл.

Файлы можно:
* выбрать через диалог;
* перетащить в область загрузки;
* открыть из карточки заявки;
* удалить, если у вас есть на это право.

h2. Полезные поля в форме заявки

h3. На что нужен бюджет

Коротко опишите суть заявки человеческим языком.

h3. Обоснование

Обязательное поле. Здесь важно объяснить, зачем нужен бюджет.

h3. Детали заявки

Необязательное поле для:
* ссылок;
* описания предмета закупки;
* контекста по задаче.

h3. Указать связанные заявки

Свернутый блок, который можно раскрыть, если заявка связана с другими заявками.

h2. Особые сценарии

h3. 1. Конкурсное задание

Для этого типа заявки есть два блока:
* Внутренние специалисты
* Подрядчики

По каждой строке можно указать:
* имя;
* цех;
* часы;
* прямые затраты.

Что важно:
* сумма заявки считается автоматически по прямым затратам;
* если для строки включена *Валидация не требуется*, HOD ее не получает;
* если валидация нужна, заявка сначала идет к руководителю нужного цеха.

h3. 2. AI-инструмент/подписка

Для этой категории по умолчанию используется источник:
* Квоты на AI-инструменты

Если источник неизвестен, можно выбрать:
* Я не знаю

h3. 3. Сервисы/транзиты для клиентов

Для этой категории доступны дополнительные поля:
* Сумма отгрузки без НДС
* Сумма отгрузки с НДС
* Коэффициент транзита
* Дата отгрузки
* Когда платят нам
* Способ оплаты

Коэффициент транзита считается автоматически.

h2. Статусы заявки

h3. Черновик

Заявка еще не отправлена на согласование.

h3. Ждет валидации цеха

Актуально для конкурсных заявок, если по специалистам нужна проверка HOD.

h3. Ожидает согласования

Заявка находится у согласующих ролей.

h3. Согласовано

Все нужные согласования получены.

h3. Требуется оплата

Автор передал заявку в оплату, дальше работает BUH.

h3. Запланирована оплата

BUH указал сумму и дату следующего платежа.

h3. Частично оплачено

Оплата идет несколькими частями.

h3. Оплачено

Оплата завершена. После этого автор обычно закрывает заявку.

h3. Заявка закрыта

Работа по заявке завершена.

h3. Отменена

Заявка больше не участвует в процессе.

h2. Что делает автор после согласования

Если по заявке нужна оплата:
# Откройте карточку заявки.
# Нажмите *Передать в оплату*.

Если оплата не требуется:
# Откройте карточку заявки.
# Нажмите *Принять без оплаты*.

h2. Как работает оплата для BUH

После передачи в оплату бухгалтер может:
* задать сумму оплаты;
* задать дату следующего платежа;
* зафиксировать полную оплату;
* зафиксировать частичную оплату;
* указать дату фактической оплаты;
* указать курс валюты;
* указать ID затрат в финплане.

h3. Полная оплата

Если заявка оплачена целиком:
# BUH открывает карточку заявки.
# При необходимости меняет сумму оплаты.
# Указывает дату фактической оплаты.
# Нажимает *Оплачено*.

h3. Частичная оплата

Если заявка оплачивается по частям:
# BUH задает или корректирует общую сумму оплаты.
# Нажимает *Запланировать следующий платеж*.
# Когда часть оплачена, указывает сумму этой части и дату оплаты.
# Нажимает *Частично оплачено*.
# При необходимости снова планирует следующий платеж.
# Когда остаток закрыт, нажимает *Оплачено*.

Если BUH меняет сумму оплаты относительно исходной суммы заявки, автор получает письмо.

h2. Комментарии, изменения и таймлайн

В карточке заявки доступны вкладки:
* Заявка
* Изменения
* Таймлайн

h3. Изменения

Показывают:
* что изменили;
* кто изменил;
* когда именно это произошло.

h3. Таймлайн

Содержит события:
* создание заявки;
* решения согласующих;
* передачу в оплату;
* планирование платежей;
* частичные оплаты;
* письма;
* загрузку файлов.

h2. Когда заявка может уйти на повторное согласование

Это возможно, если после запуска процесса меняются важные поля, например:
* сумма;
* источник финансирования;
* обязательные согласующие;
* другие ключевые параметры маршрута.

h2. Что важно помнить

* у разных категорий разные поля формы;
* не все заявки проходят одинаковый маршрут;
* суммы можно вести и без НДС, и с НДС;
* для оплаты и траншей важны даты;
* закрытую заявку можно открыть заново;
* если вы не уверены в источнике финансирования, можно выбрать *Я не знаю*.
