AURUM
Краткая инструкция для пользователей сервиса
Дата фиксации: 27.03.2026

==================================================
1. ЗАЧЕМ НУЖЕН AURUM
==================================================

Aurum — внутренний сервис AGIMA для создания, согласования и сопровождения заявок на затраты и инвестиции.

Через Aurum можно:
- создать заявку;
- отправить ее на согласование;
- передать заявку в оплату;
- отследить статус;
- увидеть историю изменений, комментарии, письма и таймлайн.

==================================================
2. ОСНОВНЫЕ РАЗДЕЛЫ
==================================================

Мои заявки
- все заявки, которые вы создали;
- тут можно фильтровать, сортировать, искать по номеру;
- отсюда можно открыть заявку, передать в оплату или закрыть.

На согласовании
- рабочий раздел для согласующих;
- если от вас требуется решение, заявка появится здесь.

Все заявки
- доступно не всем ролям;
- показывает расширенный список заявок, которые вы имеете право видеть.

Профиль
- ваши данные и роли.

==================================================
3. КАК СОЗДАТЬ ЗАЯВКУ
==================================================

1. Нажмите "Новая заявка".
2. Выберите тип заявки.
3. Выберите источник финансирования.
4. Заполните поля формы.
5. Можно:
- сохранить как черновик;
- сразу отправить на согласование.

Если заявка уже создана, ее можно редактировать.

==================================================
4. ТИПЫ ЗАЯВОК
==================================================

Сейчас доступны:
- Welcome-бонус
- Подарки
- Конкурсное задание
- Закупка сервисов
- Неформальное мероприятие
- Совместный мерч

У разных типов заявки форма немного отличается.

==================================================
5. ЧТО ВАЖНО ПРО КОНКУРСНОЕ ЗАДАНИЕ
==================================================

У конкурсного задания есть два сценария.

Сценарий 1. Без специалистов
- заполняется как обычная заявка;
- сумма указывается напрямую;
- дальше заявка идет обычным согласующим.

Сценарий 2. Со специалистами
- добавляются специалисты;
- по каждому специалисту указываются:
  - имя;
  - цех;
  - часы;
  - прямые затраты.
- общая сумма заявки считается автоматически как сумма прямых затрат по специалистам.

Важно:
- если вы не знаете прямые затраты, заявку все равно можно отправить;
- тогда сначала ее увидит руководитель нужного цеха;
- только после валидации цехами заявка пойдет обычным согласующим.

Подсказка в форме:
- возле поля прямых затрат есть текст:
  - "Уточните у руководителя цеха или отправьте на заполнение"

==================================================
6. КАК ПОНЯТЬ СТАТУС ЗАЯВКИ
==================================================

Новая заявка
- черновик, еще не отправлен.

Ждет валидации цеха
- актуально для конкурсных заданий со специалистами;
- сначала нужно, чтобы HOD провалидировал часы и прямые затраты.

Ожидает согласования
- заявка уже у согласующих.

Согласовано
- все нужные согласования получены.

Требуется оплата
- автор передал заявку в оплату, дальше работает BUH.

Запланирована оплата
- бухгалтер указал дату оплаты.

Частично оплачено
- оплата идет частями.

Оплачено
- заявка оплачена, автору нужно ее закрыть.

Заявка закрыта
- процесс по заявке завершен.

Отменена
- заявка отменена и больше не обрабатывается.

==================================================
7. ЧТО ДЕЛАТЬ ПОСЛЕ СОГЛАСОВАНИЯ
==================================================

Если по заявке нужна оплата:
1. откройте заявку;
2. нажмите "Передать в оплату".

Если оплата по счету не требуется:
1. откройте заявку;
2. нажмите "Принять без оплаты".

==================================================
8. ЧТО ДЕЛАТЬ ПОСЛЕ ОПЛАТЫ
==================================================

Когда бухгалтер переведет заявку в статус "Оплачено":
- вы получите письмо;
- в карточке заявки увидите подсказку о необходимости закрыть заявку.

Дальше нужно:
- открыть заявку;
- нажать "Закрыть заявку".

Если закрыли по ошибке:
- можно нажать "Открыть заново";
- заявка вернется в предыдущий статус.

==================================================
9. ЕСЛИ ВЫ СОГЛАСУЮЩИЙ
==================================================

Если вы NBD / COO / CFD / HOD / BUH:
- основной рабочий экран для вас — "На согласовании".

Там показываются заявки, где от вас ждут действия.

Для обычных согласующих:
- согласовать;
- отклонить;
- оставить комментарий.

Для HOD:
- провалидировать часы и прямые затраты по специалистам своего цеха.

Для BUH:
- запланировать оплату;
- отметить частичную оплату;
- отметить оплату;
- заполнить ID затрат в Финплане;
- указать курс валюты при необходимости.

==================================================
10. КОММЕНТАРИИ И ФАЙЛЫ
==================================================

Комментарии
- доступны тем, кто видит заявку;
- показывают автора, дату и время;
- комментарий можно редактировать, пока на него не ответили.

Файлы
- к большинству заявок можно прикладывать файлы;
- есть drag-and-drop;
- можно загружать несколько файлов сразу;
- есть предпросмотр для PDF и изображений.

Важно:
- в конкурсном задании файловый блок сейчас не показывается.

==================================================
11. ИСТОРИЯ И ТАЙМЛАЙН
==================================================

В карточке заявки доступны вкладки:
- Заявка
- Изменения
- Таймлайн

Изменения
- показывает, что меняли, когда и кем.

Таймлайн
- показывает все важные события:
  - создание;
  - согласования;
  - оплату;
  - письма;
  - комментарии;
  - загрузку файлов;
  - повторное согласование.

==================================================
12. ЧТО ДЕЛАТЬ, ЕСЛИ ЧТО-ТО НЕПОНЯТНО
==================================================

В интерфейсе есть всплывающие подсказки:
- на номер заявки;
- на категорию;
- на клиента / получателя;
- на сумму;
- на даты.

Если возникли проблемы:
- внизу сервиса есть контакт поддержки;
- можно написать @Natarom.

==================================================
13. КОРОТКИЙ ЖИЗНЕННЫЙ ЦИКЛ ЗАЯВКИ
==================================================

Обычная заявка:
Черновик -> Согласование -> Согласовано -> Передано в оплату / Принято без оплаты -> Оплачено -> Закрыто

Конкурсное задание со специалистами:
Черновик -> Валидация цехов -> Согласование -> Согласовано -> Передано в оплату / Принято без оплаты -> Оплачено -> Закрыто

==================================================
14. ЧТО ВАЖНО ПОМНИТЬ
==================================================

- не все типы заявок требуют одинаковые поля;
- не все заявки сразу уходят к обычным согласующим;
- если меняются важные поля, заявка может уйти на повторное согласование;
- если вы не знаете прямые затраты по конкурсному заданию, это нормально — их может уточнить HOD;
- закрытую заявку можно открыть заново.
